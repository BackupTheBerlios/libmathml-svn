attributes in MathML

In MathML attributes are generally inherited. This means that if an attribute was set in a parent, this value is also there for all child elements of the element in which the value was set. The description of inheritance behavior can be found here.

For some attributes, the inheritance rule does not apply. This means that if it's value was set in a parent that was not <mstyle>, the value does not affect the behavior of the child elements. An example for this is the attribute <linethickness from the element <mfrac>.

Implementation in libmathml

In libmathml, each document has an array with attributes. This array is stored in a private (for the library) class Attributes. It provides convenience functions such as background(), which return a reference to a MathColor object to the current value for the background color.

When laying out or painting a document, the attributes object is passed to each child attribute. Each child has two arrays with element specific attributes. One of these arrays is the array with global defaults and the other array is a local array with parse values from the attributes in the current document.
